## 1. 회원가입 페이지 만들기

### 화면 구성
- 경로: `/signup`
- 레이아웃: 독립 페이지 (Header 없음)
- 카드 중앙 정렬, 배경색 연보라 (#F5F0FF)

### 입력 필드

| 필드 | 타입 | 비고 |
|------|------|------|
| 아이디 | text | 중복확인 버튼 포함 |
| 이메일 | email | |
| 이름 | text | |
| 생년월일 | select | 년/월/일 각각 |
| 성별 | toggle | 남성/여성 |
| 패스워드 | password | 8자 이상 |
| 패스워드 확인 | password | |
| 핸드폰 번호 | text | 통신사 prefix select + 번호 입력 |
| 주소 | text | 읽기 전용, 주소 찾기 버튼 |
| 상세주소 | text | |

### 버튼 및 링크
- 회원가입 버튼: 폼 제출
- 하단 로그인 링크: `/login` 이동

### 유효성 검사
- 아이디: 중복확인 완료 후 제출 가능
- 패스워드: 8자 이상
- 패스워드 확인: 패스워드와 일치 여부 확인
- 필수 입력 항목 미입력 시 버튼 비활성화

### API 연동
- 아이디 중복확인: `GET /api/members/check-id?id={id}`
- 회원가입 제출: `POST /api/members/signup`
- 주소 찾기: 카카오 우편번호 서비스 연동

### 완료 후 동작
- 회원가입 성공 시 `/login` 페이지로 이동

## 기능
  ### 주소검색
  - 주소검색은 다음 우편번호 서비스 api를 이용하여 만든다
    필요한 라이브러리는 설치할것
  - 주소를 검색한 후 선택하면 주소 input 창에 입력되도록 한다

  ### 패스워드 확인
   - 패스워드는 패스워드 확인과 입력 값이 일치해야 한다
   - 실시간으로 비교해서 겨로가를 보여준다.
   - 패스워드와 패스워드 확인이 일치해야 회원가입이 가능하다.

   ### 필수값처리
   - 상세주소를 제외하고 모두 필수 값
   - 유효성 체크를 통해서 입력되지 않거나 유효값이 입력되지 않은경우
     입력창 하단에 유효성 체크처리

  ## 5. api를 이용한 기능 구현
- api 도메인 : https://sell-books.shop
### 5.1 아이디 중복체크 
- endpoint : /api/v1/users/check-id
- method : GET방식
- parameter: userId='admin'
### 5.2 회원가입
- endpoint : /api/v1/users/signup
- method : POST
- data 양식
```
{
    "userId"(필수) : "admin", 
    "email"(필수)  : "aaa@naver.com",
    "passwd"(필수) : "1234",
    "name"(필수)  : "홍길동",
    "gender"(필수) : "남자",
    "birthDate"(필수) : "1990-01-01",
    "phone"(필수)  : "010-3333-4444",
    "postalCode"(필수) : "03434",
    "address"(필수) :   "서울시 마포구 서강로",
    "addressDetail" : "꿈마을아파트 110동 1101호"
}
```
### 5.3  구현유의 사항
- 아이디 중복체크를 하지 않으면 경고창 출력. 회원가입 불가 
- 아이디 중복체크 후 아이디를 변경하여 입력하면 중복체크를 다시 하도록 한다. 
- 회원가입 성공/실패를 출력하고, 성공하면 로그인 페이지로 이동한다.
